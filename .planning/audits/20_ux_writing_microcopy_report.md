## Findings

### 🟠 HIGH-001: "Заказов пока нет" — мёртвый тупик без призыва к действию

**Описание:**
`orders/page.tsx` (строка 221): при 0 заказах пользователь видит `<p>Заказов пока нет</p>` — текст без CTA, без иллюстрации, без объяснения. Это классический Dead End (тупик), нарушающий Nielsen #10 "Help and Documentation".

По данным Userpilot (2025): SaaS-дашборды с проработанными Empty States имеют на **40% выше** Activation Rate у новых пользователей.

**Сравнение:**
| Текущее | Рекомендуемое |
|---------|-------------|
| "Заказов пока нет" | Иллюстрация + "Вы ещё не делали заказов. Начните продвижение за 2 минуты!" + кнопка "🚀 Создать первый заказ" |

**Затронутые файлы:** `src/app/dashboard/orders/page.tsx`, строка 221.

### 🟠 HIGH-002: Toast-ошибки в RetryPaymentModal — технический жаргон

**Описание:**
`RetryPaymentModal.tsx`, строки 48-54:
- `'Шлюз не вернул ссылку на оплату'` — слово "шлюз" непонятно 90% пользователей.
- `'Сетевая ошибка'` — технический термин.
- `'Ошибка при генерации платежа'` — "генерация" звучит механически.

**Научное обоснование:** Fogg B=MAT — если Ability (понимание) упала, Behavior (повторная попытка) не произойдёт. Человек закроет вкладку.

**Сравнение:**
| Текущее | Рекомендуемое |
|---------|-------------|
| "Шлюз не вернул ссылку на оплату" | "Не удалось создать ссылку для оплаты. Попробуйте ещё раз через минуту." |
| "Сетевая ошибка" | "Проблема с интернет-соединением. Проверьте связь и попробуйте снова." |
| "Ошибка при генерации платежа" | "Платёжная система временно недоступна. Попробуйте позже или выберите другой способ оплаты." |

**Затронутые файлы:** `src/components/orders/RetryPaymentModal.tsx`, строки 48-54.

### 🟡 MEDIUM-001: Кнопка "Оплатить 57₽" — нет эмоционального фрейминга

**Описание:**
`SmartOrderForm.tsx` (строка 552): кнопка говорит `Оплатить {totalPriceFormatted} ₽ →`. Это функционально, но не использует Framing Effect (Тверский/Канеман).

**Сравнение:**
| Текущее | Рекомендуемое (варианты) |
|---------|------------------------|
| "Оплатить 57.81 ₽ →" | "Запустить продвижение • 57.81 ₽" (action-oriented) |
| | "Получить 1000 подписчиков за 57.81 ₽" (value-oriented) |

Фрейм "Запустить продвижение" превращает расход в инвестицию (Framing Effect).

### 🟡 MEDIUM-002: Hardcoded цвета в SuccessContent — Dark Mode нарушен

**Описание:**
`SuccessContent.tsx` использует hardcoded цвета в 4 местах:
- Строка 122: `bg-amber-50 border-amber-200` (verifying circle)
- Строка 161: `bg-amber-50 border border-amber-200` (delay hint)
- Строка 207: `bg-emerald-50 border-emerald-100` (confirmed circle)
- Строка 291: `bg-rose-50 border-rose-100` (error circle)

Эти цвета не адаптированы для Dark Mode и нарушают AGENTS.md правило: "НИКОГДА не используй inline цвета".

**Рекомендация:** Заменить на semantic tokens или conditional dark: variants (`bg-amber-50 dark:bg-amber-950/30`).

---

### 🟡 MEDIUM-003: "Нет результатов." — минималистичная копия без помощи

**Описание:**
`command-menu.tsx` (строка 70): `<CommandEmpty>Нет результатов.</CommandEmpty>`. Не предлагает альтернативных действий. По исследованиям Userpilot: "No Results Found" — одно из 4 типов пустых состояний, требующее Actionable Recovery.

**Рекомендация:** "Ничего не найдено. Попробуйте изменить запрос или перейти в раздел [Заказы / Каталог]."

---

### 🟡 MEDIUM-004: "Ошибка" без объяснения в MobileOrderList

**Описание:**
`MobileOrderList.tsx` (строки 240-243): при статусе ERROR показывается заголовок "Ошибка" и сырой `{selectedOrder.error}` — строка из API провайдера на английском. Пользователь видит что-то вроде "Ошибка: link is invalid" — смесь русского и английского, без инструкции.

**Рекомендация:**
Обернуть `error` в человеко-читаемый маппер:
```
"link is invalid" → "Указанная ссылка недействительна. Проверьте URL и создайте новый заказ."
"insufficient funds" → "На балансе провайдера недостаточно средств. Мы перезапустим заказ автоматически."
```
