# План завершения MVP Админ-панели v3.0 (Остаточные работы)

Этот документ детализирует оставшиеся задачи для завершения рефакторинга SMMplan до версии 3.0, основываясь на выполненных фазах 1 и 2 (Навигация, Каталог, RBAC, Маркетинг и Финансы).

---

## Epic 1: Advanced Analytics (Глубокая аналитика)
**Цель:** Дать владельцу платформы инструменты для оценки рентабельности на уровне отдельных услуг и когорт пользователей.

*   **Task 1.1: Дашборд Рентабельности (Profitability Dashboard)**
    *   Агрегация данных из `Order` и `Payment` для расчета чистой прибыли по каждой `Service` и `Category`.
    *   Выявление "убыточных" или низкомаржинальных услуг (где `providerCost` съедает прибыль).
*   **Task 1.2: Анализ LTV Клиентов (Customer Cohorts)**
    *   Графики удержания пользователей и распределения выручки по топ-10% клиентов (китам).
    *   Интеграция графиков (Recharts) с поддержкой темной/светлой темы HeroUI.
*   **Task 1.3: Экспорт и Отчеты**
    *   Генерация CSV/PDF отчетов по финансовым показателям для бухгалтерии.

---

## Epic 2: Support & Tickets (Система поддержки)
**Цель:** Ускорить работу саппорта, внедрить умные ответы и интегрировать тикеты с финансовым модулем.

*   **Task 2.1: Редизайн интерфейса тикетов**
    *   Перевод списка тикетов (`/admin/tickets`) на `DataTable` с фильтрами по статусу, приоритету и источнику (WEB/TELEGRAM).
    *   Интерфейс чата внутри тикета с предпросмотром истории заказов клиента.
*   **Task 2.2: AI-Ассистент (Smart Replies)**
    *   Добавление кнопки "Сгенерировать ответ" с использованием LLM (на основе контекста заказа и шаблонов `SupportTemplate`).
    *   Управление базой знаний саппорта (сохранение удачных ответов в шаблоны).
*   **Task 2.3: Интеграция EscrowGuard в тикеты**
    *   Добавление виджета "Компенсация" прямо в интерфейс тикета, чтобы саппорт мог в 2 клика вернуть средства на баланс (с учетом суточных лимитов `supportLimitCents`).

---

## Epic 3: Provider Sync Hardening (Интеграция с VexBoost)
**Цель:** Обеспечить 100% надежность статусов заказов и защиту от рассинхрона балансов.

*   **Task 3.1: Интеграция Webhook**
    *   Разработка эндпоинта (например, `/api/webhooks/vexboost`) для получения PUSH-уведомлений о смене статусов заказов от провайдера (избавляет от тяжелого Cron-polling'а).
    *   Валидация подписей (HMAC/Secret) для защиты эндпоинта.
*   **Task 3.2: Обработка Partial / Cancelled**
    *   Автоматический перерасчет `remains` и начисление возвратов на `balance` пользователя с обязательной записью в `LedgerEntry`.
*   **Task 3.3: Quarantine & Price Spikes (Защита от скачков цен)**
    *   Автоматическая приостановка услуги (статус `isQuarantined = true`), если провайдер поднял цену более чем на заданный `quarantineThreshold` (например, 20%).
    *   Уведомление администратора о необходимости ручного аппрува новой цены.

---

## Epic 4: Production Deployment Prep (Подготовка к релизу)
**Цель:** Обеспечить безопасный, быстрый и отказоустойчивый деплой новой архитектуры.

*   **Task 4.1: CI/CD Pipeline (GitHub Actions)**
    *   Настройка пайплайна: Lint (ESLint Flat Config), Typecheck (TypeScript 5.7+), Tests (Vitest).
    *   Сборка Docker-образа и пуш в Registry.
*   **Task 4.2: Оптимизация Docker**
    *   Обновление `Dockerfile` под Next.js 16 Standalone output для минимизации размера образа.
    *   Настройка `docker-compose.yml` для продакшена (PostgreSQL connection pooling, Redis для rate-limiting).
*   **Task 4.3: Security Audit**
    *   Проверка настроек Rate Limit (Next.js middleware / Nginx).
    *   Проверка ротации сессий и токенов авторизации.
    *   Финальное тестирование уязвимостей RBAC.
