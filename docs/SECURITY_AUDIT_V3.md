# Архитектурный Аудит Безопасности (Smmplan)

**Статус:** Выполнен
**Инструмент:** ANTIGRAVITY ORCHESTRATOR_v3.0

## 1. Резюме аудита
В ходе проверки были проанализированы Server Actions, API-маршруты и NPM-зависимости. В целом, проект использует современные паттерны защиты (Server-side guards вроде `enforcePageRole`, `verifySession` и `requireAdmin`). Однако выявлено несколько векторов атак, которые требуют исправления.

## 2. Найденные уязвимости (Tier B Analysis)

### 2.1. Уязвимость: Открытый кэш-инвалидатор (DoS вектор)
**Уровень угрозы:** Medium
**Расположение:** `src/app/api/debug/route.ts`
**Описание:** Маршрут принимает параметр `revalidate` и вызывает `revalidateTag(revalidate)` без какой-либо проверки авторизации.
**Доказательство (Proof):** Любой неавторизованный пользователь может отправить `GET /api/debug?revalidate=cms-list` и сбросить кэш приложения, что открывает путь к атаке типа Denial of Service (DoS) на базу данных из-за частых регенераций страниц.
**Решение:** Обернуть вызов в `enforcePageRole(["ADMIN", "OWNER"])` или удалить этот файл в Production окружении.

### 2.2. Уязвимости в NPM Зависимостях (XSS)
**Уровень угрозы:** Moderate
**Расположение:** `postcss` (зависимость `next`)
**Описание:** Найдены 2 уязвимости средней степени тяжести (GHSA-qx2v-qp2m-jg93). Существует вектор XSS-атаки через неэкранированный тег `</style>` в `postcss` при обработке стороннего CSS.
**Доказательство (Proof):** Вывод `npm audit --json` показал уязвимость `<8.5.10` для `postcss`.
**Решение:** Запустить `npm audit fix` или обновить Next.js до версии, которая включает `postcss@8.5.10+`.

### 2.3. Потенциальная утечка данных в `debug/route.ts`
**Уровень угрозы:** Low (информационный)
**Расположение:** `src/app/api/debug/route.ts`
**Описание:** Возвращает `cookieStore.getAll()` и сессионные токены. Хотя это данные самого пользователя, выставление наружу механизма сессий в виде JSON на проде может облегчить работу злоумышленнику при Session Hijacking (если есть активный XSS вектор).

## 3. Проверенные надежные узлы (Best Practices)
*   [Evidence: `src/actions/admin/providers/crud.ts`] Управление провайдерами обернуто в строгий guard `requireStaffPermission`, ключи API шифруются перед сохранением (`VaultService.encrypt()`).
*   [Evidence: `src/actions/user/top-up.action.ts`] Проверка пополнения баланса имеет защиту от спама: `RateLimitService.check('topup:${session.userId}', 5, 300)`.
*   [Evidence: `src/actions/admin/content.ts`] Защита от системных слагов при создании контента.

## 4. Рекомендации по исправлению (State 5)
1.  **Удалить или защитить `api/debug/route.ts`.**
2.  **Провести обновление зависимостей (`npm audit fix`).**
3.  **Внедрить Content Security Policy (CSP)** в `next.config.ts`, чтобы окончательно нивелировать XSS векторы (особенно в блоке `dangerouslySetInnerHTML`, который используется в `JsonLd.tsx`).
