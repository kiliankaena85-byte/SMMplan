# Requirements: SMMplan_lite

**Defined:** 2026-04-16
**Core Value:** Бесшовный и моментальный опыт покупки (Smart URL Matching) для пользователя и абсолютная финансовая прозрачность (Бухгалтерия) для владельца.

## v1 Requirements

### Authentication
- [ ] **AUTH-01**: Аккаунт пользователя создается автоматически (или привязывается сессия) при первом оформлении заказа.
- [ ] **AUTH-02**: Авторизация в личный кабинет происходит через Magic Link отправленный на Email.
- [ ] **AUTH-03**: Сессия пользователя безопасно сохраняется (JWT / Cookies).

### Smart Ordering (Frontend)
- [ ] **ORD-01**: Пользователь вставляет URL; система вызывает парсер, который возвращает соцсеть и разворачивает список релевантных услуг.
- [ ] **ORD-02**: Цены на витрине и в каталоге отображаются и рассчитываются за 1 единицу (с красивым округлением).
- [ ] **ORD-03**: Пользователь может оформить Массовый заказ через специальное текстовое поле.
- [ ] **ORD-04**: Пользователь может настроить плавную накрутку (Drip-feed).
- [ ] **ORD-05**: Базовый ручной каталог услуг работает как fallback для неизвестных ссылок.

### Order Processing & Billing (Backend)
- [ ] **CORE-01**: Интеграция шлюзов YooKassa и Crypto-эквайрингов. Система полностью поглощает их комиссии за свой счет.
- [ ] **CORE-02**: Обработчик заказов (Order Processor) резервирует баланс, отправляет API-запрос провайдеру (Vexboost, Stream-promotion) и фиксирует ProviderOrderID.
- [ ] **CORE-03**: Воркер (Sync) опрашивает статусы провайдеров. Строгая State Machine исключает двойные возвраты.
- [ ] **CORE-04**: Статус "Partial" по API автоматически пересчитывает остаток и оформляет частичный возврат на баланс в ЛК.
- [ ] **CORE-05**: При ошибке заказа у провайдера система переводит его в статус Canceled и делает возврат (Failover отключен).

### Financial & Accounting
- [ ] **FIN-01**: Финансовый дашборд админа показывает Gross Revenue, Net Profit, COGS (затраты API) и комиссии шлюзов.
- [ ] **FIN-02**: Налоговый калькулятор автоматически резервирует (отображает) настраиваемый % от выручки в счет уплаты налогов (УСН/Прибыль РФ 2026).

### Support & RBAC Administration
- [ ] **SUP-01**: Ролевая модель: Admin, Support, SEO, Investor с жестким разграничением прав.
- [ ] **SUP-02**: Омниканальная тикетная система объединяет обращения с ЛК на сайте, Email и Telegram-бота поддержки.
- [ ] **SUP-03**: Контекстный интерфейс поддержки: внутри тикета видны последние заказы пользователя, транзакции, и доступен переход к ним.
- [ ] **SUP-04**: Система макросов, черновиков, бан/варн функционал и внутренние комментарии для "коллег".
- [ ] **SUP-05**: Встроенная база знаний (инструкции/шпаргалки) для новых саппортов.
- [ ] **SUP-06**: Новая надежная Headless CMS-подсистема для редактирования статических страниц и FAQ.

### Marketing
- [ ] **MKT-01**: Система промокодов (фиксированные и процентные, с лимитами).
- [ ] **MKT-02**: Накопительная программа лояльности для автоматических скидок оптовикам.
- [ ] **MKT-03**: Мультиязычность: архитектура и роутинг с поддержкой локалей (RU/EN).

## v2 Requirements

### Provider Extensions
- **EXT-01**: Инструментарий B2B API для подключения реселлеров к ресурсам SMMplan_lite.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Геймификация (NPS, Бейджи) | Излишнее усложнение версии "Lite". |
| Oauth авторизация (Telegram, VK, Google) | Риски блокировок или нестабильности в РФ 2026, используем только Magic Link. |
| Telegram-бот для заказов | Бот используется исключительно как транспорт для саппорта. Заказы только в Web. |
| Автоматический API Failover | Отключен для обеспечения стабильности стейт-машины и исключения коллизий. |
