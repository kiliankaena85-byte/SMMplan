## ✅ Что уже хорошо

Landing page содержит базовый уровень Social Proof:
- ✅ `TrustBar.tsx` — счётчики: "2,000,000+ Заказов выполнено"
- ✅ `Reviews.tsx` — блок отзывов клиентов с аватарками
- ✅ Login page — мини-статистика: "99% Выполнено"

**Проблема:** Всё это на Landing. Когда пользователь попадает в Dashboard и открывает форму заказа — **весь Social Proof исчезает**. Именно в момент принятия решения о покупке (checkout) Trust Signals отсутствуют полностью.

## Findings

### 🔴 CRITICAL-001: Чекаут полностью лишён Trust Signals — 0 элементов доверия

**Описание:**
`SmartOrderForm.tsx` — 608 строк формы заказа. Grep по `Visa|Mastercard|МИР|PCI|SSL|shield|безопасн|Защищ` = **0 результатов** (кроме иконок CreditCard/Wallet/Bitcoin, которые — функциональные, а не trust-элементы).

По Baymard Institute (2023): **18% пользователей бросают корзину**, потому что "не доверяли сайту свои данные карты". Это 2-я по частоте причина после "скрытых расходов".

Checkout без Trust Signals — это как магазин без кассовой ленты: деньги берут, но ощущение безопасности = 0.

**Научное обоснование:**
- Baymard Institute, "Checkout UX Research" (2023): 18% abandonment from trust issues.
- Fogg Trust Triangle: отсутствует ось Surface Trust (визуальные сигналы).
- Cialdini — Authority: нет логотипов платёжных систем = нет аргумента авторитета.

**Затронутые файлы:** `src/components/orders/SmartOrderForm.tsx`, область рядом с кнопкой "Оплатить" (строки 494-554).

**Рекомендация:**
Под кнопкой "Оплатить" добавить строку Trust Badges:
```
🔒 Безопасная оплата • Visa / Mastercard / МИР • Powered by ЮKassa
```
Или SVG-иконки платёжных систем в строке, как делают Stripe, Shopify, Tinkoff.

### 🟠 HIGH-001: Нет гарантий в карточке услуги

**Описание:**
Grep по `гарант|warranty|refund|возврат|защит` в `SmartOrderForm.tsx` = **0 результатов**. Карточка услуги (строки 251-290) показывает: название, бейдж, скорость, цену. Но не показывает **гарантийный срок** и **тип гарантии**.

Пользователь не знает: "Если подписчики отпишутся — мне вернут деньги?" Этот вопрос остаётся без ответа до момента, когда пользователь уже потеряет деньги и напишет в поддержку.

**Научное обоснование:**
- Risk Reversal (Cialdini): Гарантия снимает барьер "а вдруг не сработает?".
- Каждый невысказанный страх = фрикция на пути к покупке.

**Рекомендация:**
В карточке услуги под "Скорость" добавить: `🛡️ Гарантия 30 дней` (если у услуги warranty > 0). Данные уже есть в БД — нужно только пробросить в UI.

### 🟠 HIGH-002: Social Proof пропадает после Landing

**Описание:**
`TrustBar.tsx` и `Reviews.tsx` рендерятся только на Landing. В Dashboard (**авторизованная зона**, где пользователь принимает решение о покупке) — **0 Social Proof элементов**.

Пользователь заходит на лендинг, видит "2,000,000+ заказов", читает отзывы, решает зарегистрироваться. После регистрации попадает в чистый Dashboard — и все аргументы "за" исчезают.

**Рекомендация:**
Добавить мини-TrustBar в header Dashboard или в footer `SmartOrderForm`:
```
✅ 2,000,000+ выполненных заказов • ⚡ Среднее время запуска: 3 мин
```

### 🟠 HIGH-003: Нет индикатора "как работает услуга" — непрозрачность

**Описание:**
Пользователь видит "Premium Подписчики Telegram — 57₽ / 1000". Но НЕ видит:
- Откуда берутся подписчики? (бот, реальные аккаунты?)
- Что произойдёт после оплаты? (через сколько начнут приходить?)
- Будут ли видны в статистике канала?

Без этих ответов мозг переключается на System 2 и начинает гуглить "smmplan отзывы" — пользователь уходит со страницы.

**Рекомендация:**
Под описанием услуги добавить кнопку "ℹ️ Как это работает?" → аккордеон с FAQ по данному типу услуги.

---

### 🟡 MEDIUM-001: Нет публичной Status Page

**Описание:**
Когда сайт упал или провайдер сломался, пользователь не имеет способа узнать "это у меня проблема или у них?". Status Page (как у Stripe, GitHub, AWS) — стандарт SaaS-сервисов уровня B2B.

**Рекомендация:**
Простая страница `/status` с зелёными/красными индикаторами для:
- Веб-сайт: 🟢
- Платёжная система: 🟢
- Провайдеры: 🟢/🟡/🔴

---

### 🟡 MEDIUM-002: Нет "живых" уведомлений о покупках (Live Activity Feed)

**Описание:**
Многие SMM-панели используют "живую ленту" в углу экрана:
> "👤 Алексей только что заказал 5000 подписчиков Telegram"

Это максимально мощный триггер Social Proof (Cialdini). Но внедрять осторожно — если данные фейковые, это нарушает 11_legal_compliance (Dark Patterns).

**Рекомендация:** Внедрять только на основе реальных данных из таблицы `Order` (анонимизированных).
