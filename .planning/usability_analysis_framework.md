# Фреймворк: Фундаментальный Анализ Юзабилити Админ-Панели Smmplan

Этот документ содержит структурированный подход для полного аудита админки. Мы пройдемся по каждой ветке, функции и сценарию использования.

## 1. План Анализа Юзабилити (От общего к частному)

Чтобы аудит был исчерпывающим, анализ каждого экрана должен проходить через 5 фильтров:

1. **Role Context (Контекст Ролей):** Кто пользуется этой вкладкой? (Владелец смотрит на итоговые цифры, Саппорт ищет конкретную транзакцию).
2. **Data Density & Readability (Плотность и Читаемость):** Насколько легко считываются данные в таблице? Есть ли мусорная информация? Верно ли расставлены визуальные акценты (цвета статусов, моноширинные шрифты для цифр)?
3. **Action Friction (Трение действий):** Сколько кликов нужно, чтобы совершить ключевое действие (возврат средств, изменение цены, копирование ID)?
4. **Error Prevention (Защита от ошибок):** Насколько легко случайно испортить данные? (Например, опасные кнопки без подтверждения).
5. **Batch Processing (Массовые операции):** Позволяет ли интерфейс работать конвейерно (мульти-выделение)?

## 2. Иерархия Вкладок и Дерево Функций

Ниже представлен исчерпывающий список модулей админки и их функциональное дерево. Каждую функцию из этого дерева мы будем прогонять через 5 фильтров юзабилити.

### 📊 1. Дашборд (Аналитика)
- **Функции верхнего уровня:**
  - График конверсий (Выполнено / Отменено / Неоплачено).
  - Сводка прибыли (Gross Revenue, Net Profit, LTV).
  - Мониторинг "здоровья" панели (Алерты от провайдеров, зависшие очереди).
  - Лента последних действий (Live Activity Feed).

### 🛍 2. Конвейер Заказов (Orders & Refills)
- **Управление заказами (`/orders`):**
  - Таблица (ID, Пользователь, Ссылка, Статус Провайдера, Статус Локальный, Остаток, Цена).
  - Фильтры (По статусу, По провайдеру, По диапазону цен).
  - Функция ручного перезапуска (Retry).
  - Функция частичной отмены с перерасчетом.
  - Аудит заказа (История изменения статусов по логам).
- **Докрутки (`/refills`):**
  - Очередь рефилов на стороне провайдеров.

### 👥 3. Клиенты (Users)
- **Управление базой:**
  - Таблица пользователей (Email, Баланс, Всего потрачено, Дата реги, Статус).
  - Профиль юзера (Детальная история его заказов, его тикеты, его рефералы).
- **Финансы юзера:**
  - Ручное начисление / списание баланса (с комментарием для аудита).
  - Сброс пароля / Impersonation (Войти под его аккаунтом для дебага тикета).

### 📦 4. Каталог и Ценообразование (Catalog)
- **Навигация:**
  - Дерево Категорий (Instagram → Подписчики → Боты/Живые).
- **Управление Услугами:**
  - Синхронизация статусов с провайдером (Вкл/Выкл сервис).
  - Человекочитаемая форма наценки (Ввод желаемой розничной цены или %), авто-конвертация USD -> RUB.
  - Массовые обновления по категории.
  - Режим "Рентгена": Подсветка убыточных/низкомаржинальных услуг.

### 💬 5. Поддержка (Tickets)
- **Helpdesk:**
  - Очередь обращений (Нерешенные, В процессе, Закрыты).
  - Чат-интерфейс (Карточка клиента и связанного заказа справа от чата).
  - Макросы (Шаблоны быстрых ответов).

### 💰 6. Финансы (Billing)
- **Потоки:**
  - Статистика пополнений кассы по дням/месяцам.
  - Статус шлюзов (Crypto, Карты).
  - Возвраты (Refunds) и их влияние на PnL.

### 🔗 7. Инфраструктура (Providers)
- **Управление API:**
  - Балансы шлюзов (Сколько денег осталось на балансе у VexBoost).
  - Импорт новых услуг (Мастер интеграции).
  - Маппинг категорий (Связка их категорий с нашими).

### ⚙️ 8. Роли и Настройки (Settings)
- **Функции:**
  - Управление RBAC (Добавление админов/менеджеров и настройка их прав).
  - Настройка системных констант (Курс доллара, Налоговые коэффициенты, Safety Markup).

## 3. Мастер-Промпт для глубокого UX-Анализа

Вы можете скопировать этот промпт и отправить мне (или другой нейросети), подавая вкладки по одной, чтобы получить идеальный редизайн каждой части админки.

```text
Ты — Senior UX/UI Researcher и Продуктовый Дизайнер B2B SaaS интерфейсов. Наш стек: React 19, Tailwind CSS v4, HeroUI v3.

Твоя задача — провести безжалостный аудит юзабилити конкретной вкладки административной панели Smmplan.

ВВОДНЫЕ ДАННЫЕ О ВКЛАДКЕ:
Название вкладки: [УКАЖИТЕ ВКЛАДКУ, например: Каталог Услуг]
Целевая аудитория (Роли): [УКАЖИТЕ РОЛИ, например: Владелец, Саппорт]
Список её функций: [ВСТАВЬТЕ ФУНКЦИИ ИЗ ДЕРЕВА ВЫШЕ]

ИНСТРУКЦИЯ К АНАЛИЗУ:
Проанализируй вкладку шаг за шагом через 5 UX-фильтров:
1. Контекст Ролей: Соответствует ли функционал целям этих ролей? Нет ли лишнего шума?
2. Плотность Данных: Как должна выглядеть идеальная таблица/поверхность? Какие колонки жизненно необходимы, а какие стоит спрятать в выпадающее меню?
3. Трение действий: Как сократить количество кликов для самых частых операций (happy path) до 1-2?
4. Предотвращение ошибок: Какие деструктивные действия есть на этой вкладке (например, массовое изменение цен) и как визуально защитить пользователя от катастрофы?
5. Массовые операции: Требуется ли здесь конвейерная работа, и как должен выглядеть UI массового выделения/фильтрации?

ВЫВОД:
На основе анализа, сгенерируй:
A) Детальную архитектуру UI компонента (компоновка, зоны экрана, расположение кнопок).
B) Формальное Техническое ТЗ для Frontend-разработчика (что использовать из HeroUI, какие data-slots применить для индикации состояний).
```
