## Findings

### 🟠 HIGH-001: F-Pattern — Статус заказа находится на 5-й колонке, а не в начале

**Описание:**
Таблица заказов (`orders/page.tsx`, строки 95-101) имеет порядок колонок: `ID → Услуга → Ссылка/Кол-во → Сумма → Статус → Дата`. По F-Pattern глаз сканирует первую горизонтальную полосу (заголовки), затем спускается вниз по левому краю. **Статус** — самый важный для пользователя элемент (ответ на вопрос "Готов ли мой заказ?") — находится на 5-й позиции, куда взгляд доходит последним.

**Научное обоснование:**
- Nielsen Norman Group (2006): "F-Shaped Pattern For Reading Web Content" — пользователи тратят 80% внимания на первые 2 колонки.
- Идеальный порядок для task-driven дашборда: `Статус → Услуга → Кол-во → Сумма → Дата`.

**Затронутые файлы:** `src/app/dashboard/orders/page.tsx`, строки 94-101.

**Рекомендация:**
Переместить колонку "Статус" на 1-ю или 2-ю позицию. Или добавить цветную точку-индикатор в колонку ID (например, зелёный/оранжевый/красный кружок перед `#numericId`).

### 🟠 HIGH-002: Карточки услуг не имеют hover-эффекта подъёма (0 микро-анимаций)

**Описание:**
Grep по `hover:shadow|hover:scale` в `SmartOrderForm.tsx` = **0 результатов**. Карточки услуг (строки 252-290) используют только `transition-all duration-200` для смены цвета рамки. Нет ощущения "подъёма" при наведении (hover:shadow-md, hover:-translate-y-0.5).

**Научное обоснование:**
- Дофаминовое подкрепление: микро-анимации при наведении дают мозгу мгновенную обратную связь "да, ты навёл на интерактивный элемент". Это снижает неопределённость и увеличивает уверенность в клике.
- Material Design Guidelines: "Surface elevation changes communicate interactivity."

**Затронутые файлы:** `SmartOrderForm.tsx`, строки 258-262.

**Рекомендация:**
```diff
- 'ring-1 ring-slate-200/60 bg-white hover:ring-sky-300 hover:bg-slate-50 shadow-sm'
+ 'ring-1 ring-border bg-card hover:ring-primary/50 hover:bg-muted hover:shadow-md hover:-translate-y-0.5 shadow-sm'
```

### 🟠 HIGH-003: Кнопка "Оплатить" не является самым контрастным элементом на мобилке

**Описание:**
На десктопе кнопка "Оплатить" (строка 546) находится в правой панели (sticky) — это хорошо. Но на мобилке она прилипает к нижней панели (fixed bottom bar, строка 495-501), которая визуально "сливается" с остальным интерфейсом: `bg-card border-t border-border`.

Кнопка CTA должна быть **самым визуально тяжёлым элементом на экране** (Von Restorff Effect / Isolation Effect): элемент, отличающийся от остальных, запоминается и получает внимание.

**Научное обоснование:**
- Von Restorff Effect (1933): изолированный стимул запоминается лучше.

**Рекомендация:**
Усилить контраст floating bar: добавить `shadow-[0_-10px_40px_rgba(0,0,0,0.12)]` или градиентный fade сверху (как в Uber Eats / Delivery Club).

### 🟡 MEDIUM-001: Цена услуги — моноширинный шрифт, но нет визуального "якоря" на старую цену

**Описание:**
Цена (строка 286) отображается как `font-black text-slate-900 tabular-nums font-mono`. Это хорошо для читаемости, но нет зачёркнутой "старой цены" или процента скидки. Без визуального сравнения пользователь не понимает, получает ли он выгоду.

**Рекомендация:** Если есть данные об исходной цене провайдера или ранее действовавшей цене — показать зачёркнутым шрифтом рядом (`line-through text-muted-foreground text-xs`).

---

### 🟡 MEDIUM-002: Отсутствие Whitespace-границы между категориями и списком услуг

**Описание:**
Табы категорий (строка 196) и список услуг (строка 245) разделены только `space-y-4` контейнера. Для чёткого Gestalt разделения (Принцип Proximity: "элементы, расположенные рядом, воспринимаются как группа") рекомендуется разделитель или увеличение gap. Пользователь может не осознавать, что табы и список — это два разных функциональных блока.

**Рекомендация:**
Добавить визуальный разделитель `<div className="border-b border-border my-4" />` между табами и списком.

---

### 🟡 MEDIUM-003: Отсутствует "пустое состояние" с призывом к действию на странице заказов

**Описание:**
Если у пользователя 0 заказов, рендерится пустая таблица. Пустое состояние (Empty State) — упущенная точка конверсии. По Nielsen's Heuristic #1 ("Visibility of System Status") пользователь должен понимать, что система работает и предлагает следующий шаг.

**Рекомендация:**
При `orders.length === 0` показать иллюстрацию + текст "У вас пока нет заказов. Начните с первого!" + кнопка "Создать заказ".
