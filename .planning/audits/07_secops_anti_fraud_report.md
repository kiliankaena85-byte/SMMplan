## ✅ Что уже хорошо
1. **Защита от Race Conditions (Баланс):** При оплате с внутреннего баланса используется `MutexManager.withLock` (строка 218 в `checkout.ts`) вместе с транзакциями БД. Это полностью исключает возможность конкурентного списания средств в "минус" (DDoS-вектор кражи баланса).

## Findings

### 🔴 CRITICAL-001: Rate Limit Bypass через IP Spoofing (Подделка IP)

**Описание:**
В файле `rate-limit.service.ts` (строки 23-24) IP пользователя определяется так:
```typescript
const forwardedFor = headerList.get("x-forwarded-for");
const ip = forwardedFor ? forwardedFor.split(",")[0] : "127.0.0.1";
```
Злоумышленник может отправить HTTP-заголовок `X-Forwarded-For: 1.1.1.1` (или любой другой случайный IP) напрямую в свой запрос. Прокси добавит реальный IP в конец списка: `1.1.1.1, 8.8.8.8, 123.45.67.89`. Метод `split(",")[0]` возьмет **первый** IP (поддельный).

**Научное обоснование:**
- *IP Spoofing:* Злоумышленник может менять поддельный IP в каждом запросе, полностью обходя лимиты (`maxHits = 15`). Это позволяет брутфорсить промокоды, спамить заказами и ложить базу данных `AWAITING_PAYMENT` записями.

**Рекомендация:**
При использовании Cloudflare нужно брать `cf-connecting-ip`. Для других прокси (Nginx) нужно использовать `x-real-ip`.
Код должен быть:
```typescript
const ip = headerList.get("cf-connecting-ip") || headerList.get("x-real-ip") || headerList.get("x-forwarded-for")?.split(",")[0] || "127.0.0.1";
```

### 🟠 HIGH-001: Отсутствие поведенческой биометрии (Bot Mitigation)

**Описание:**
Форма заказа (`checkout.ts`) защищена только Rate Limit'ом по IP. Нет ни скрытой капчи (Cloudflare Turnstile, reCAPTCHA v3), ни проверки скорости ввода данных, ни фингерпринтинга браузера.

**Вектор атаки:**
Конкуренты могут запустить скрипт на Selenium/Puppeteer, который будет оформлять заказы каждую секунду (используя резидентные прокси для обхода Rate Limit). База данных переполнится "мусорными" заказами, а ЮKassa может заблокировать магазин за аномальное количество неоплаченных счетов.

**Рекомендация:**
Интегрировать Cloudflare Turnstile в форму заказа. Валидировать токен в `checkoutAction` перед запуском транзакции.
