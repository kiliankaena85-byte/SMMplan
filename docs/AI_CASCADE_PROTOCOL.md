# 🧠 Протокол: Каскад ИИ-Скиллов (AI Cascade Protocol)

**Версия:** 1.0.0
**Статус:** Утверждено к исполнению (Active Protocol)

Этот документ описывает "Каскад Скиллов" — топологию взаимодействия ИИ-агентов в проекте Smmplan. Запрещено использовать агента-исполнителя (Хирурга) напрямую для сложных задач в обход Каскада.

---

## 🌊 Уровни Каскада (The Waterfall of Agents)

Процесс разработки движется сверху вниз. Переход на следующий уровень возможен только после успешного завершения предыдущего (Gate Passed).

### Level 1: Стратегия и Оркестрация (Intake)
*Когда применять:* Бизнес-идея, новая крупная фича, архитектурный сдвиг.
1. **`gsd-chief-of-staff` (Глава Аппарата):** Точка входа. Опрашивает Директоров (Security, CFO), взвешивает риски, формирует Executive Summary.
2. **`gsd-prompt-engineer` (Аналитик):** Если Chief of Staff дал зеленый свет, Аналитик берет интервью у Пользователя (USER) и превращает идею в жесткое техническое задание (ТЗ).

### Level 2: Проектирование и Риски (Blueprint)
*Когда применять:* Перед написанием плана (PLAN.md).
1. **`gsd-ui-architect`:** Генерирует JSON-контракт для фронтенда (состояния, пропсы). Гарантирует соблюдение `ENGINEERING_STANDARDS.md`.
2. **`gsd-research-autopsy`:** Исследует подводные камни (Edge Cases, совместимость библиотек). Генерирует Матрицу Рисков (Risk Matrix P×I).

### Level 3: Исполнение (Execution)
*Когда применять:* Утвержденный PLAN.md готов.
1. **`gsd-surgeon` (Хирург):** Автономный исполнитель. Пишет код строго по спецификациям Аналитика и Архитектора. Выполняет атомарные коммиты.
2. **`gsd-ui-designer`:** Узкий специалист по Tailwind 4. Подключается Хирургом, если нужно сверстать сложный премиальный интерфейс.
3. **`gsd-db-migrate`:** Обертка над базой данных. Вызывается Хирургом для безопасного наката Prisma-миграций.

### Level 4: Валидация и Приемка (Validation Gates)
*Когда применять:* Код написан, нужно делать коммит/релиз.
1. **`gsd-preflight-check` (Надзиратель):** Физический Gate (AST парсинг, tsc, eslint). Если ловит "use server" в page.tsx или отсутствие auth-гарда — отправляет Хирурга в Refactor Loop.
2. **`gsd-scrum-master`:** Проверяет соблюдение Definition of Done (DoD).
3. **`gsd-verify-work`:** Интерактивное UAT-тестирование с Пользователем.

### Level 5: Ретроактивный Аудит (Analytics & Red Team)
*Когда применять:* По расписанию или по требованию для контроля качества.
1. **`gsd-omni-audit`:** Оркестратор аудиторов.
2. **Спецназ Аудита:**
   - `gsd-analytics-sec` (Атакует наши экшены, ищет IDOR).
   - `gsd-analytics-cfo` (Проверяет расчеты маржи).
   - `gsd-analytics-debt` (Ищет мусорный код и TODO).

---

## 🚨 Правила Каскадирования (Cascade Rules)

1. **Anti-Bypass Rule:** Хирург (`gsd-surgeon`) имеет право отказаться писать код и вернуть задачу на Level 1, если сочтет ТЗ размытым.
2. **Conflict Resolution:** Если `gsd-ui-designer` хочет добавить inline-цвета для "красоты", а `gsd-preflight-check` это запрещает — приоритет всегда у проверяющего гейта (Compliance > Aesthetics). Дизайнер обязан найти семантический токен из дизайн-системы.
3. **Context Reset:** Между переходом агентов с Level 3 на Level 4 оркестратор (GSD) обязан очистить контекст (сбросить логи чата), чтобы агент-валидатор не попал под влияние "галлюцинаций" Хирурга.
