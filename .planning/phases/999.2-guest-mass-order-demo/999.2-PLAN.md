## 2. Критерии приемки (User Acceptance Criteria)

### UAT-1: Автопереключение ввода
- **Действие:** Пользователь вставляет в единое поле ввода одну ссылку.
- **Ожидаемый результат:** Обычный флоу одиночного заказа.
- **Действие 2:** Пользователь нажимает Enter и пишет вторую строку, либо вставляет текст с разделителями строк.
- **Ожидаемый результат 2:** Текстовое поле превращается в многострочный `textarea`, включается массовый режим, отображается таблица-превью калькуляции строк.

### UAT-2: Заботливый перехват почты
- **Действие:** Пользователь вставляет почту `client@example.com` в поле ввода ссылки.
- **Ожидаемый результат:** Кнопка «Поехали» меняется на «Запомнить почту?». Рядом выводится нежный текст: *«Похоже, вы ввели email-адрес. Использовать его для связи и отправки чеков?»*.
- **Действие 2:** Пользователь кликает «Использовать».
- **Ожидаемый результат 2:** Поле `email` заполняется значением `client@example.com`, поле ссылки очищается, показывается красивый зеленый чек-марк.

### UAT-3: Бесшовная оплата гостя
- **Действие:** Гость нажимает «Оформить массовый заказ» с заполненной почтой.
- **Ожидаемый результат:** Показывается модал подтверждения: *«Мы создадим личный кабинет на почту client@example.com. Всё верно?»*. При согласии выполняется Server Action, создается аккаунт в БД, заказы привязываются к нему, и происходит перенаправление на платежный шлюз.
