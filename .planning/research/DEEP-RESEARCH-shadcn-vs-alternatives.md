## 1. Концепция: "Copy-Paste" против "NPM Dependency"

Современные библиотеки делятся на два лагеря:
- **NPM Зависимости (Material UI, Ant Design, Mantine):** Вы устанавливаете `npm install @mui/material`. Код компонентов скрыт от вас глубоко в `node_modules`. Вы общаетесь с ними через строгий API (например, `<Button variant="contained" color="primary">`).
- **Headless Copy-Paste (Shadcn/ui):** Вы запускаете команду `npx shadcn-ui@latest add button`. Система **копирует** исходный код React-кнопки прямо в вашу папку `src/components/ui/button.tsx`. Вы становитесь полноправным владельцем этого кода.

## 2. Сравнительный анализ для Admin Dashboard (Next.js 16)

### 🥇 Shadcn/ui (Выбор Tech Lead для Smmplan)
- **Философия:** Полный контроль над исходным кодом. Компоненты строятся на базе Tailwind CSS и Radix UI (движок доступности).
- **Плюсы:** 
  - Нет "вендор-лока" (vendor lock-in) на стороннюю библиотеку. 
  - Идеальная работа с Next.js App Router и Server Components (так как код лежит в вашем проекте, компилятор легко его анализирует).
  - Минимальный размер бандла (вы скачиваете только те компоненты, которые реально используете).
  - ИИ-совместимость (Vercel v0 генерирует код именно в формате Shadcn).
- **Минусы (Risk Audit):** Вы сами должны поддерживать код. Если в логике компонента (например, DatePicker) найдут баг, вам придется обновлять файл `date-picker.tsx` руками. Для огромных корпоративных команд на 100+ человек это сложно. Для нас — это идеальная свобода.

### 🥈 Mantine (Сильнейший конкурент)
- **Философия:** "Батарейки в комплекте". Огромный набор готовых хуков (например, `useForm`, `useMediaQuery`) и UI компонент.
- **Плюсы:** Невероятно быстрый старт. Очень дружелюбна к разработчику (Developer Experience).
- **Минусы:** Более тяжелый бандл. Вы привязаны к их API тем (Theming API), который сложно "сломать" или сильно изменить под абстрактный пиксель-дизайн.

### 🥉 Ant Design (AntD)
- **Философия:** Тяжелый корпоративный энтерпрайз (родом из Alibaba).
- **Плюсы:** Имеет готовые невероятно сложные сущности (например, таблицы с вложенной группировкой, древовидные селекторы).
- **Минусы:** ОЧЕНЬ тяжелый бандл. "Квадратный" китайский корпоративный стиль, который мучительно сложно перекрашивать и делать современным (Glassmorphism, градиенты и т.д.). Для Smmplan — будет выглядеть как устаревшая 1С.

### 🏅 Material UI (MUI)
- **Философия:** Догматичный Material Design от Google.
- **Плюсы:** Непробиваемая надежность.
- **Минусы:** Устаревший визуальный язык (Android 2016-2020 годов). CSS-in-JS движок historically плохо дружил с Next.js Server Components.

### 🏅 HeroUI (ранее NextUI)
- **Философия:** Apple-подобная эстетика прямо из коробки.
- **Плюсы:** Очень красивые анимации, свечения (glow effects) и тени по умолчанию.
- **Минусы:** Сложно сделать кастомный дизайн. Вы воюете с их предустановленными Tailwind-классами, если хотите сделать что-то нестандартное.

## 3. Вывод (Decision Log: Smmplan Architectural Strategy)

Почему для **Smmplan** нам нужен именно **Shadcn/ui**:

1. **Единый движок стилей (Tailwind CSS 4.x):** У нас в проекте уже установлен Tailwind. Shadcn не тянет сторонние "движки тем" вроде CSS-in-JS или Emotion. Все цвета и отступы мы задаем через родной Tailwind, что гарантирует молниеносный рендеринг в Next.js.
2. **Отношение к бизнесу:** Мы делаем платформу для реселлерского бизнеса. Нам может понадобиться специфическая таблица заказов с прогресс-барами внутри ячеек (которая работает с нашей Prisma-базой). Сделать это в AntD или MUI — значит воевать с их документацией. Сделать это в Shadcn — это просто отредактировать *наш собственный* файл таблицы в `src/components/ui/table.tsx`.
3. **Современная Эстетика:** В отличие от скучного Enterprise (AntD), Shadcn позволяет нам внедрить элементы Magic UI (анимации Framer) поверх стандартных таблиц без конфликтов библиотек.

**[x] VALIDATED:** Использование Shadcn/ui для модернизации админ-панели (Admin 2.0) является наиболее безопасным и гибким архитектурным решением для Next.js App Router в 2026 году.
