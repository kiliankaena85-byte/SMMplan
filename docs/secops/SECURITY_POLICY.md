# Smmplan Lite: Политика Безопасности (OWASP ASVS 5.0 & PCI DSS)

## 1. Введение
Эта политика устанавливает стандарты безопасности, которые должны безукоризненно соблюдаться при разработке любой новой функции в экосистеме Smmplan. Цель — защита пользовательских данных, финансовых балансов и инфраструктуры.

## 2. Финансовые Системы и Транзакции (PCI DSS Approach)
Smmplan обрабатывает балансы в копейках (`Int`), поэтому:
1.  **Strict Idempotency:** Любой входящий вебхук оплаты обязан иметь ID проверки на уникальность. Отклонение дубликатов происходит в `O(1)`.
2.  **Atomic Upserts & Transactions:** Переводы реферальных балансов, зачисление средств и оплаты (`checkoutAction`) выполняются только через `db.$transaction()` с сериализованным уровнем изоляции. Математическая проверка исключает уход баланса в минус (`if (balance < 0) throw`).
3.  **No Raw Card Data:** Мы не храним CVV или PAN. Вся платежная информация проходит через шлюзы-партнеры (YooKassa / CryptoBot).

## 3. Защита Интерфейсов и API
1.  **Rate Limiting:** Интегрирован `RateLimitService` (Redis-based) для пресечения брутфорса Magic Link и Mass Orders. 
2.  **Auth Guards:** Для вызова `Server Actions` из админ-панели встроен `requireAdmin()` middleware. Любой доступ без сессии отсекается.
3.  **XSS & Data Escaping:** Категорический отказ от `dangerouslySetInnerHTML`. Все данные Telegram-ботов и пользовательских ссылок очищаются и пропускаются через Zod.

## 4. Защита от Уязвимостей Среды
*   Запрет на использование `any` типов (строгий TS 5.7+).
*   ESLint 10 Flat Config содержит правила `@typescript-eslint/no-unsafe-argument` и плагины `eslint-plugin-security`. 

Любое нарушение данных правил влечет за собой отклонение Pull Request на этапе CI/CD.
